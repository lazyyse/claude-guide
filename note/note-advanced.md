## 02｜Claude Projects 完全活用

### Projects とは何か

Claude Projects（プロジェクト機能）は、**Claude に「背景知識」と「一貫した指示」を持たせる**機能です。通常のチャットでは会話が終わるたびに記憶がリセットされますが、Projects を使うと：

- カスタム指示（常に守ってほしいルール）を設定できる
- ドキュメント・資料を「知識ベース」として読み込める
- 同じプロジェクト内では文脈が保持される
- チームメンバーと共有できる（有料プランの一部機能）

### カスタム指示の設定方法

Projects の「カスタム指示」欄に入力した内容は、そのプロジェクト内のすべての会話に自動的に適用されます。

**設定例：営業チーム向けプロジェクト**

```
【基本設定】
あなたは当社（株式会社〇〇）の営業支援AIです。

【会社情報】
・業種：〇〇
・主要製品：〇〇
・ターゲット顧客：〇〇

【トーン・スタイル】
・ビジネス文書はフォーマルな敬語を使用
・箇条書きを積極的に活用
・数字・固有名詞は必ず確認を促す

【禁止事項】
・競合他社の批判はしない
・確認できていない情報を断言しない
```

この設定を一度行えば、毎回「丁寧に」「敬語で」などと指示する必要がなくなります。

### 知識ベースの活用

Projects にはドキュメントをアップロードして、Claude に「読ませる」ことができます。

**活用例：**
- 自社製品の仕様書・FAQ → 問い合わせ回答の下書きに
- 社内ガイドライン・規定 → 文書作成時のチェックに
- 過去の提案書・成功事例 → 新規提案の参考に
- 競合分析レポート → 差別化ポイントの整理に

**注意：** アップロードするドキュメントに機密情報・個人情報が含まれる場合は、社内のAI利用ポリシーを確認してから使用してください（詳細はセクション07）。

### Projects 活用の実践ステップ

1. **claude.ai にログイン → 左サイドバーの「Projects」→「新規作成」**
2. **プロジェクト名を設定**（例：「営業支援」「採用業務」「マーケ文書」）
3. **カスタム指示を入力**（上記テンプレートを参考に）
4. **必要なドキュメントをアップロード**
5. **プロジェクト内でチャット開始**

## 03｜他ツール連携の実例5本

### 連携の基本的な考え方

ClaudeはAPIを通じて他ツールと連携できますが、ノンエンジニアが手軽に始めるなら**Zapier・Make（旧Integromat）などのノーコード連携ツール**を使うのが最速です。

ここでは実際の業務でよく使われる5つの連携パターンをステップ付きで解説します。

---

### 連携①：Gmail → Claude → 下書き自動生成

**シナリオ：** メールを受信したら、Claudeが返信の下書きを作成してGmailの下書きフォルダに保存する

**ツール：** Gmail ＋ Zapier ＋ Claude API（または Zapier の AI機能）

**ステップ：**
1. Zapier で「Gmail - New Email」をトリガーに設定
2. フィルター：特定の件名・送信者のみに絞る（全メールは処理しすぎになる）
3. Claude（またはZapier AI）へ：「以下のメールへの返信下書きを〇〇のトーンで作成してください：{{メール本文}}」
4. Gmail の「Create Draft」アクションで下書きとして保存
5. 人間が確認・修正して送信

**ポイント：** 完全自動送信にせず、必ず人間が確認するフローにすること。

---

### 連携②：Notion → Claude → 議事録の整理・要約

**シナリオ：** Notionに貼り付けた会議メモを、Claudeが整理して決定事項・アクションアイテムを抽出する

**ツール：** Notion ＋ Zapier ＋ Claude

**ステップ：**
1. Notion に「会議メモ」テンプレートを用意（タグ：「要整理」）
2. Zapier で Notion の「要整理」タグ付きページ追加をトリガーに
3. Claudeへ：「以下の会議メモから、①決定事項、②継続検討事項、③アクションアイテム（担当者・期限）を抽出してください：{{Notionページ本文}}」
4. Notionの別ページ（「整理済み議事録」）に出力を保存
5. タグを「整理済み」に更新

**手動版：** Zapierなしで手動でも十分効果的。会議後にメモをコピペしてClaudeに依頼するだけで5分が1分になる。

---

### 連携③：Excel/Googleスプレッドシート → Claude → 分析レポート

**シナリオ：** 売上データをClaudeに渡し、週次レポートの文章を自動生成する

**ツール：** Googleスプレッドシート ＋ Google Apps Script（GAS）＋ Claude API

**ステップ（手動版）：**
1. スプレッドシートの該当範囲をCSVでコピー
2. Claudeへ貼り付けて：「以下の週次売上データを分析し、①前週比、②好調な商品・地域、③懸念点、④来週への示唆の4点でレポートを作成してください」
3. 出力をそのままレポートやSlackに貼り付け

**自動化版（GAS）：** Google Apps Script でスプレッドシートのデータをClaude APIに送信し、結果を別シートに書き出すことも可能（Claude Code 入門ページで詳しく解説）。

---

### 連携④：Slack → Claude → FAQへの自動回答

**シナリオ：** Slackの特定チャンネルに質問が投稿されたら、Claudeが社内FAQをもとに回答する

**ツール：** Slack ＋ Zapier ＋ Claude Projects（FAQ知識ベース入り）

**ステップ：**
1. Claude Projects に社内FAQドキュメントをアップロード
2. Zapier で「Slack - New Message in Channel」をトリガーに（例：#ai-support チャンネル）
3. Claude へ：「以下の質問に社内FAQをもとに回答してください。FAQにない場合は『担当者に確認します』と答えてください：{{質問文}}」
4. Zapierの「Slack - Send Message」で同チャンネルに返信

**注意：** 誤った情報を自動返信するリスクがあるため、最初は「下書き回答をSlackに投稿して人間が確認・送信する」フローから始めることを推奨。

---

### 連携⑤：Zapier Webhook → Claude → 定期レポート生成

**シナリオ：** 毎週月曜9時に前週のデータをClaudeに集計させ、Slackに週次レポートを送信する

**ツール：** Zapier（スケジュールトリガー）＋ データソース ＋ Claude ＋ Slack

**ステップ：**
1. Zapier の「Schedule by Zapier」で毎週月曜9時をトリガーに設定
2. データソース（Googleスプレッドシート・Notion・外部API）から前週データを取得
3. Claudeへデータを渡してレポート文章を生成
4. Slackの指定チャンネルに自動投稿

**コスト感：** Zapierの無料プランは月100タスクまで。上記のような自動化は有料プラン（月約2,000円〜）が現実的。

## 04｜時短効果シミュレーター

### 自分の業務への当てはめ方

AI活用の効果を「なんとなく便利」で終わらせず、具体的な数字で把握することが社内展開の説得力につながります。

以下の計算式で自分の業務の時短効果を試算してみてください。

**計算式：**

```
週間時短時間 = Σ（各業務の削減時間 × 週の実施回数）
年間時短時間 = 週間時短時間 × 50週
```

### 業務別・AI活用前後の時間比較

※【ここに時短効果の比較表を画像として挿入】

### 試算例：営業担当者（週40時間勤務）の場合

```
・提案書作成：60分 → 20分（▲40分）× 週3回 = ▲120分/週
・メール作成：5分/通 → 2分/通（▲3分）× 週20通 = ▲60分/週
・議事録整理：30分 → 5分（▲25分）× 週3回 = ▲75分/週
・リサーチ：60分 → 20分（▲40分）× 週2回 = ▲80分/週

合計：▲335分/週（約5.6時間）
年間：▲280時間
```

週5時間以上の削減は、月に約22時間分の余裕に相当します。この時間を戦略的な業務・顧客との関係構築・スキルアップに使えます。

### Claudeへの試算依頼テンプレート

```
私の主な業務と週当たりの作業時間を以下に示します。
Claudeを活用した場合の時短効果を試算し、
優先的に導入すべき業務の順番を提案してください。

【業務リスト】
・〇〇：週〇時間
・〇〇：週〇時間
・〇〇：週〇時間

【AI活用経験】：〇〇（未経験 / 初心者 / ある程度使っている）
【使用環境】：Claude無料プラン / Pro / その他
```

## 05｜ノーコード業務自動化の実例

### 「自動化」と「AI活用」の違い

「自動化」と聞くとプログラミングが必要なイメージがありますが、ここではコードを一切書かずに実現できる業務効率化に絞って解説します。

**コード不要でできること：**
- Zapier・Make を使った複数ツール連携
- Google Apps Script のテンプレート（Claudeが書いてくれる）をコピペで使う
- Notion・Airtable の自動化機能
- Microsoft Power Automate（Office 365 ユーザー向け）

### 実例①：週次レポートの半自動化

**現状：** データ収集30分 ＋ 文章作成60分 = 90分/週
**改善後：** データ収集15分 ＋ Claude に貼り付けて生成5分 ＋ 確認・修正10分 = 30分/週

**Claudeへの依頼プロンプト：**

```
以下のデータをもとに、社内向け週次レポートを作成してください。

【データ】：（スプレッドシートから貼り付け）

【レポートの形式】：
・冒頭：1段落の総括（3文以内）
・主要指標：前週比で重要な変化を3点
・好調要因・懸念点：各2点
・来週の注目ポイント：2点
・全体：400字以内

【読者】：部門マネージャー層
【トーン】：簡潔・客観的・数字重視
```

### 実例②：採用書類の初期スクリーニング支援

**現状：** 履歴書・職務経歴書を1通20分で読む × 50通 = 1,000分（約17時間）
**改善後：** Claudeに評価ポイントを整理させてから読む → 1通10分 × 50通 = 500分（約8時間）

**Claudeへの依頼プロンプト：**

```
以下の職務経歴書を読み、当社の採用基準に照らして評価してください。

【採用基準】：
・必須：〇〇の実務経験3年以上
・歓迎：チームリード経験、〇〇ツールの使用経験
・カルチャーフィット：〇〇な方

【職務経歴書】：（貼り付け）

出力：
①必須要件の充足度（充足 / 不足 / 不明）
②歓迎要件の該当有無
③強みと懸念点（各2点）
④総合所見（3文）
⑤推奨アクション：面接 / 書類選考通過 / 見送り
```

### 実例③：SNS運用の効率化

**現状：** 週3投稿 × 30分/投稿 = 90分/週
**改善後：** 月1回まとめてClaudeに12投稿分を生成 → 確認・修正で月60分 = 15分/週

**Claudeへの依頼プロンプト：**

```
以下の条件で、X（旧Twitter）の投稿文を12本（4週分・週3本）作成してください。

【アカウントの目的】：〇〇業界のビジネスパーソン向けに有益な情報を発信
【ターゲット】：〇〇
【トーン】：〇〇
【投稿テーマ（12本分）】：
1週目：〇〇に関するtips
2週目：〇〇の事例紹介
3週目：〇〇のQ&A
4週目：〇〇の考え方

各投稿は140字以内、ハッシュタグ2〜3個付き。
曜日（月・水・金）と投稿番号も記載してください。
```

## 06｜社内展開ロードマップ

### なぜ「個人活用」から「組織活用」が難しいのか

Claudeを個人で使い始めると「便利だ」とすぐわかります。しかし組織で展開しようとすると壁にぶつかります。

- 「情報漏洩が怖い」という声
- 「使い方がわからない」という声
- 「何に使えばいいか具体的にイメージできない」という声
- 管理職の理解が得られない

これらの壁を乗り越えるためのロードマップを解説します。

### フェーズ1：個人実績をつくる（1〜2ヶ月）

まず自分の業務でClaude活用の実績をつくります。数字で示せる成果が社内展開の最強の説得材料になります。

**やること：**
- 自分の業務でClaude活用を試す（中級編テンプレートを活用）
- 時短効果・品質向上を記録する（セクション04の計算式を使用）
- 「使ってよかった事例」を3件以上ストックする

### フェーズ2：小さなチームで試す（2〜3ヶ月）

個人実績をもとに、直属の上司・同僚2〜3名に使ってもらいます。

**やること：**
- フェーズ1の実績を共有
- 業務別のプロンプトテンプレートを共有（中級編テンプレート30選が使える）
- 使ってみた感想・改善点を収集
- チーム固有のプロンプトを育てる

**注意：** この段階では「強制」しない。使いたい人だけ使う状態から始める。

### フェーズ3：部門展開（3〜6ヶ月）

チームでの実績をもとに部門規模で展開します。

**やること：**
- 部門全体への説明会（15〜30分）を実施
- 社内ガイドライン（セクション07参照）を整備
- 部門専用の Claude Projects を作成
- 月1回の「活用事例シェア会」を開始
- 効果測定の仕組みをつくる（時短時間・品質スコアなど）

### フェーズ4：全社展開（6〜12ヶ月）

部門実績をもとに全社展開を進めます。

**やること：**
- 全社向けAI活用ポリシーの策定（セクション07のテンプレート使用）
- 全社員向けの研修（初級編をベースに社内版を作成）
- AI活用推進担当者の任命
- 定期的な効果測定・改善サイクルの確立

### 社内説明会のプレゼン構成

```
① なぜ今AIを使うのか（3分）
   - 業界・競合の動向
   - 自分たちが「使わない」コスト

② 実際に使ってみた事例（5分）
   - 自分の業務での具体例
   - 時短時間・品質変化を数字で

③ みなさんの業務に当てはめると（5分）
   - 部門ごとの活用例
   - 今日からできる第一歩

④ 安全に使うためのルール（3分）
   - 入力してはいけない情報
   - 出力の確認方法

⑤ 質疑応答（10分）
```

## 07｜情報漏洩・著作権・社内ルール整備

### AI活用の3大リスク

上級者ほど見落とすのが「リスク管理」です。使い始めると便利さが先行してリスクが後回しになりがちです。組織でAIを使う際に意識すべき3大リスクを整理します。

### リスク① 情報漏洩

**何が問題か：**
入力した情報がAIの学習データとして使われる可能性があります。Claude（Anthropic）の利用規約では、Claude.aiの無料・有料プランで入力したデータは、一定の条件下でモデルの改善に使用される場合があります。

**対策：**
- 個人情報（氏名・住所・メールアドレスなど）は入力しない
- 未公開の財務情報・M&A情報は入力しない
- 機密性の高い契約書・提案書は固有名詞を仮名に置き換えてから入力する
- 法人向けプラン（Claude for Business）ではデータの学習利用をオプトアウトできる

**実践：機密情報の仮名化テンプレート**

```
以下の文書の固有名詞（会社名・人名・金額）を
〔会社A〕〔担当者1〕〔金額X〕のように置き換えてから
Claude に入力してください。
```

### リスク② 著作権

**何が問題か：**
Claudeが生成した文章・コードの著作権の扱いは現時点でグレーな部分があります。また、Claudeが学習データに含まれる著作物に影響を受けた出力をすることがあります。

**現実的な対策：**
- 生成物は「下書き」として扱い、必ず人間が加筆・修正する
- そのまま公開・提出する前に、独自性の確認を行う
- コードの場合、ライセンスに注意が必要なOSSのコードが含まれていないか確認する

### リスク③ ハルシネーション（誤情報）

**何が問題か：**
初級編でも触れましたが、Claudeは自信を持って誤った情報を生成することがあります。組織での利用では、誰かが確認せずにそのまま使ってしまうリスクがあります。

**対策：**
- 統計・数値・法律・医療・税務の情報は必ず一次情報源で確認
- 「確認が必要な箇所には【要確認】と付記してください」とプロンプトに入れる
- 出力の確認フロー（誰が・どのタイミングで確認するか）を明文化する

### 社内AIガイドライン 雛形

以下をそのまま使って社内向けガイドラインを作成できます。

---

**【社内向け】Claude（生成AI）利用ガイドライン（ver.1.0）**

**目的**
本ガイドラインは、社員がClaudeを安全かつ効果的に業務で活用するための基準を定めるものです。

**入力してはいけない情報**
- 個人情報（顧客・社員の氏名・住所・電話番号・メールアドレス）
- 未公開の財務情報・経営情報
- 機密性「極秘」「秘」に分類される社内文書
- 取引先との守秘義務契約の対象となる情報

**入力してよい情報（社内判断で使用可）**
- 一般公開されている情報をもとにした作業
- 個人情報を仮名化した文書
- 社内の業務プロセス・手順書（機密情報を除く）

**出力物の取り扱い**
- 重要な数字・固有名詞・法的情報は必ず確認する
- そのまま社外に提出・公開する場合は上長の確認を得る

**問い合わせ先**
AI活用に関する疑問・相談：〔担当部署・連絡先〕

---

### 著作権・倫理的利用の原則

1. **差別・ハラスメントにあたる生成は行わない**
2. **フェイク情報・誤情報の生成に使わない**
3. **他者の著作物をそのまま模倣・複製させない**
4. **生成AIを使ったことを必要に応じて開示する**

## 08｜理解度チェッククイズ

以下の7問で、上級編の理解を確認してください。

**Q1.** Claude Projects のカスタム指示に設定した内容は、どのスコープで適用されますか？
→ **A. そのプロジェクト内のすべての会話に自動適用される**

**Q2.** 社内展開のロードマップで推奨している最初のステップはどれですか？
→ **A. まず自分の個人業務で実績をつくり、数字で示せる成果をストックする**

**Q3.** ClaudeのAPI連携でノンエンジニアが最も手軽に始められるツールはどれですか？
→ **A. ZapierやMakeなどのノーコード連携ツール**

**Q4.** 機密性の高い文書をClaudeに入力する際の正しい対応はどれですか？
→ **A. 固有名詞（会社名・人名・金額）を仮名に置き換えてから入力する**

**Q5.** Claudeが生成した統計・数値・法的情報の扱いとして正しいのはどれですか？
→ **A. 必ず一次情報源で確認する。確認なしにそのまま使わない**

**Q6.** 社内AIガイドラインで「入力してはいけない情報」として最も重要なのはどれですか？
→ **A. 個人情報・未公開財務情報・機密分類の社内文書**

**Q7.** 「草案はAIが生成し、最終判断は人間が行う」この分担が最も効果的な理由はどれですか？
→ **A. AIの処理速度と人間の文脈判断・責任能力を組み合わせることで、品質と効率を両立できるから**

## まとめ

この記事で学んだことを5点でまとめると：

1. **Projects で「AIに背景を覚えさせる」** — カスタム指示と知識ベースで毎回の説明が不要になる
2. **連携ツールで「繰り返し業務を自動化」** — ZapierとClaudeを組み合わせれば定型業務を大幅削減
3. **時短効果を数字で把握する** — 「なんとなく便利」から「年間〇時間削減」へ
4. **社内展開は個人実績から始める** — 強制より共感。事例が最強の説得材料
5. **リスク管理を仕組み化する** — ガイドライン整備と確認フローが組織活用の土台
