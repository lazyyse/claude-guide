## 02｜役割指定（ロールプロンプト）の技術

### なぜ役割を与えると精度が上がるのか

Claudeは「次に来る言葉として最も自然なもの」を確率的に選んで文章を生成しています。役割（ロール）を与えることで、その確率分布が大きく変わります。

「ベテラン秘書として」と指定すれば、Claudeは秘書が書くような丁寧で簡潔なビジネス文章のパターンを参照します。何も指定しなければ、汎用的な平均的な文章が返ってきます。

役割指定は**たった1行で回答品質を変える**、最もコストパフォーマンスの高い技術です。

### 役割指定の基本構文

> あなたは〇〇（役割）です。〇〇（タスク）をしてください。

これだけです。シンプルですが、〇〇の中身で大きく変わります。

### 効果的な役割の4つのパターン

**パターン①：職種・専門性を指定する**

> あなたは15年のキャリアを持つ採用人事のプロです。

職種を指定するだけでなく、「15年のキャリア」などの経験年数を加えると、より専門的な視点の回答が得られます。

**パターン②：読者・対象を指定する**

> あなたはIT未経験の50代ビジネスパーソンに説明する先生です。

誰に向けて話すかを指定すると、用語の難易度・例えの選び方が変わります。

**パターン③：スタイル・トーンを指定する**

> あなたはマッキンゼーの戦略コンサルタントとして、論理的・簡潔に回答してください。

「マッキンゼー」「BCG」「P&G」など有名企業・組織を引用すると、そのブランドに紐づくコミュニケーションスタイルが反映されます。

**パターン④：批判的・客観的な視点を持たせる**

> あなたは私のビジネスプランに懐疑的な投資家です。弱点を厳しく指摘してください。

肯定的な役割だけでなく、あえて批判的・反対側の役割を与えることで、自分では気づかない盲点を発見できます。

### 役割指定の注意点

役割指定は強力ですが、注意点もあります。

- **過剰な役割は逆効果**：「世界最高の〇〇として」など誇張しすぎると、かえって不自然な回答になる
- **役割と矛盾した指示をしない**：「子どもにわかる言葉で説明する先生として、専門用語を使って説明してください」は矛盾している
- **毎回同じ役割を使い回さない**：タスクに合わせて役割を変えることが重要

## 03｜失敗プロンプト修正道場

初級でも「悪い例・良い例」を学びましたが、中級では**実際の業務でよくある失敗パターン**を15種類取り上げ、なぜ失敗するのかと修正方法を解説します。

### 失敗パターン一覧

**失敗①：目的が抜けている**

❌ `議事録を作って`
✅ `以下の会話テキストから、決定事項・担当者・期限を抽出した議事録を箇条書きで作成してください。`

目的（何のための議事録か）と出力形式（箇条書き）を追加するだけで、使えるアウトプットになります。

---

**失敗②：対象読者が不明**

❌ `生成AIについて説明して`
✅ `生成AIを一度も使ったことがない50代の経営者に向けて、専門用語を使わず、ビジネスメリットを中心に300字で説明してください。`

誰に向けた説明かによって、言葉の選び方・例えの種類・深さが大きく変わります。

---

**失敗③：長さの指定がない**

❌ `このメールの返信を書いて`
✅ `このメールの返信を、150字以内で簡潔に書いてください。`

長さを指定しないと、Claudeは「適切だと判断した長さ」で返します。それが長すぎたり短すぎたりすることが多いです。

---

**失敗④：複数のタスクを一度に依頼する**

❌ `この企画書を要約して、問題点を指摘して、改善案も出して、プレゼン用のスライド構成も作って`
✅ タスクを分割して順番に依頼する

複数タスクを一度に依頼すると、どれも中途半端になります。「まず要約 → 次に問題点 → 最後に改善案」と順番に依頼するのが正解です。

---

**失敗⑤：禁止事項を言わない**

❌ `商品の紹介文を書いて`
✅ `商品の紹介文を書いてください。ただし、「革命的」「前例のない」などの誇張表現は使わないでください。`

望まない表現・形式・内容を明示的に禁止することで、出力の質が安定します。

---

**失敗⑥：前提情報を渡していない**

❌ `この件についてどう思いますか？`
✅ 背景・状況・関係者・制約などの文脈を先に提供する

「この件」が何かをClaudeは知りません。過去の会話でも、新しい会話では記憶がリセットされています。常に必要な文脈を含めましょう。

---

**失敗⑦：曖昧な形式指定**

❌ `わかりやすくまとめて`
✅ `以下の3点を、それぞれ2文以内で箇条書きにまとめてください：①〜、②〜、③〜`

「わかりやすく」は人によって異なります。具体的な構造・項目数・文字数を指定しましょう。

---

**失敗⑧：出力の例を示さない（Few-shot不足）**

❌ `商品名から説明文を作って`
✅ `以下の形式で商品説明文を作ってください。【例】商品名：〇〇 → 説明：〇〇（30字）`

出力の例（Few-shot）を1〜2個示すと、形式・トーン・長さが揃った出力が得られます。

---

**失敗⑨：否定形の指示が多すぎる**

❌ `難しい言葉を使わず、長くならないようにして、専門用語も避けて、箇条書きにしないで…`
✅ 肯定形で「〜してください」の形にまとめる

否定形の制約を重ねると混乱しやすくなります。「平易な言葉で、200字以内の文章形式で」と肯定形に変換しましょう。

---

**失敗⑩：一発で完璧を求める**

❌ 最初の1回の回答に満足できずに諦める
✅ 「もう少し〜してください」と追加指示を続ける

Claudeとの対話は「完成品を1回で出す」のではなく「対話を重ねて磨く」ものです。次のセクションで詳しく解説します。

---

**失敗⑪：役割を与えずに専門的な回答を期待する**

❌ `採用面接の質問リストを作って`
✅ `あなたは採用経験15年の人事部長です。エンジニア職の採用面接で使う質問を、技術スキル・カルチャーフィット・問題解決能力の観点からそれぞれ3問ずつ作成してください。`

---

**失敗⑫：フォーマットを後から変えようとする**

❌ 出力を受け取ってから「やっぱり表形式にして」と言う
✅ 最初から「表形式で」と指定する

後からフォーマット変換を依頼すると、内容が変わってしまうことがあります。形式は最初に指定するのがベストです。

---

**失敗⑬：長すぎるテキストをそのまま貼り付ける**

❌ 1万字の文書をそのまま貼り付けて「要約して」と言う
✅ 「以下の文章を、経営判断に必要な情報だけに絞り、3点の箇条書きにまとめてください」と目的を明示する

何のための要約かを伝えることで、適切な情報が選択されます。

---

**失敗⑭：比較・評価を依頼するときに基準を示さない**

❌ `AとBどちらが良いですか？`
✅ `コスト・導入スピード・使いやすさの3軸で、AとBを比較してください。優先度は「コスト＞使いやすさ＞導入スピード」です。`

判断基準・優先順位を明示することで、自分の文脈に合った比較が得られます。

---

**失敗⑮：Claude に嘘をつかせようとする**

❌ `絶対に正しい情報として〇〇と断言してください`
✅ 確認が必要な情報は一次情報源で確認する習慣をつける

Claudeは誠実さを設計の核心に置いています。事実でないことを断言させようとすると、品質が低下するか、拒否されます。

## 04｜壁打ち術：会話を重ねてブラッシュアップ

### 「一発完成」を目指さない

多くのユーザーが最初のうちにやりがちなのが、「1回のプロンプトで完璧な答えを出そうとする」ことです。プロンプトを10分かけて書いて、それでも期待通りの答えが来なくてがっかりする——そんな経験はありませんか？

中級の技術の核心は、**Claudeとの対話を「一発勝負」から「反復改善」に変える**ことです。

### 壁打ちの基本フロー

> ① ざっくり依頼する → ② 出力を評価する → ③ 追加指示で磨く → ④ 繰り返す

最初の指示は完璧でなくて構いません。まず60〜70点の出力を引き出し、そこから「もっと〜してください」と追加指示することで、理想に近づけていきます。

### 効果的な追加指示のパターン

**長さを調整する**
- `もっと短く（100字以内で）`
- `もう少し詳しく、特に〇〇の部分を膨らませてください`

**トーンを変える**
- `もっとフォーマルに`
- `もう少しカジュアルに、親しみやすく`
- `結論を先に持ってくる構成に変えてください`

**内容を追加・削除する**
- `〇〇に関する情報を追加してください`
- `〇〇の部分は不要なので削除してください`
- `具体的な数字や事例を加えてください`

**視点を変える**
- `顧客の立場から見た場合はどうなりますか？`
- `反対意見を挙げてください`
- `もっとシンプルに言い換えてください`

### 壁打ちの実例：企画書の導入文を磨く

**1回目：ざっくり依頼**
> 社内のAI活用推進プロジェクトの企画書の導入文を書いてください。

↓

**2回目：長さとトーンを調整**
> もう少し短く（150字以内）、経営層が読む前提で、投資対効果を前面に出した書き方にしてください。

↓

**3回目：具体性を追加**
> 「〇〇%の業務時間削減」など数字を含めて、説得力を高めてください。

3回の対話でほぼ完成品になります。最初から完璧なプロンプトを書こうとするより、はるかに効率的です。

### コンテキストを引き継ぐ技術

長い会話では、Claudeが最初の文脈を忘れることがあります。そのときは：

- `最初の条件（〇〇宛、〇〇のトーン）を維持したまま、〜を変更してください`
- `ここまでの内容を踏まえた上で、〜してください`

と明示的に文脈を再提示すると精度が保たれます。

## 05｜Chain of Thought・構造化プロンプト

### Chain of Thought（思考の連鎖）とは

Chain of Thought（CoT）とは、Claudeに**答えを出す前に思考過程を経由させる**技術です。

複雑な問題・多段階の判断が必要なタスクで特に効果を発揮します。

**指定方法：**
> `段階的に考えてから、最終的な答えを出してください。`
> `まず問題を整理し、その後で回答してください。`

シンプルな一文を加えるだけで、論理的な推論を経た回答が得られます。

### 構造化プロンプトとは

XMLタグやマークダウン記法を使って、プロンプトを構造化する技術です。複雑な指示を整理して渡すことで、Claude が正確に解釈しやすくなります。

**例：XML構造化**

```
<role>15年のキャリアを持つ採用人事のプロ</role>
<task>以下の職務経歴書を評価してください</task>
<criteria>
  - 技術スキルの適合度
  - リーダーシップの有無
  - カルチャーフィットの可能性
</criteria>
<output>各項目を5段階評価し、総合コメントを200字で</output>
<document>
  （職務経歴書の内容）
</document>
```

項目が明確に分かれているため、Claudeが何を・どのように処理すべきかを正確に理解できます。

### Few-shot プロンプティング

Few-shotとは、出力の**例を1〜3個示してから本番の依頼をする**技術です。

**例：**

```
以下の形式で、商品の紹介文を作成してください。

【例1】
商品名：スマートノート
紹介文：手書きの内容を自動でデジタル化。会議中のメモがそのままクラウドに保存されます。

【例2】
商品名：集中タイマー
紹介文：25分作業・5分休憩のポモドーロ技法を自動化。作業効率が平均30%向上します。

では以下の商品の紹介文を作成してください。
商品名：〇〇
```

例を見せることで、文体・長さ・構造が統一された出力が安定して得られます。

## 06｜コピペで使える業務テンプレート30選

職種・業務別に30種のテンプレートを掲載します。`【】`の部分を自分の状況に合わせて書き換えてすぐ使えます。

### 営業・提案

**① 提案書の導入文**
```
あなたは法人営業10年のベテランセールスです。
以下の条件で提案書の導入文を作成してください。

【提案先】：
【提案内容】：
【相手の課題】：
【強調したいポイント】：
【文字数】：200字以内
【トーン】：フォーマル・簡潔・課題解決を前面に
```

**② 商談後のお礼メール**
```
以下の情報をもとに、商談後のお礼メールを作成してください。

【宛先・会社名・担当者名】：
【商談日】：
【話した主な内容】：
【次のアクション（あれば）】：
【トーン】：丁寧・簡潔・関係構築を意識
【文字数】：150〜200字
```

**③ 断りの連絡**
```
以下の条件で、取引先への断りの連絡文を書いてください。
相手との関係を壊さず、今後の取引の可能性を残す表現にしてください。

【相手】：
【断る内容】：
【断る理由（簡潔に）】：
【代替案（あれば）】：
```

**④ 競合比較表の作成依頼**
```
以下の商品・サービスについて、指定の観点で比較表を作成してください。

【比較対象】：【A】、【B】、【C】
【比較観点】：価格、機能、サポート体制、導入実績
【用途】：【営業資料 / 社内検討用 / etc.】
表形式で出力し、各項目に簡単なコメントを添えてください。
```

### マーケティング・広報

**⑤ SNS投稿文（X/Twitter）**
```
以下の情報をもとに、X（旧Twitter）の投稿文を3パターン作成してください。

【告知内容】：
【ターゲット】：
【強調したいポイント】：
【文字数】：各140字以内
【トーン】：【親しみやすい / フォーマル / 専門的】
ハッシュタグも2〜3個提案してください。
```

**⑥ プレスリリースの下書き**
```
あなたは広報担当のプロです。
以下の情報をもとに、プレスリリースの下書きを作成してください。

【発表内容】：
【会社名・担当部署】：
【発表日】：
【背景・目的】：
【主な特徴・メリット（3点）】：
【問い合わせ先】：
標準的なプレスリリースの構成（見出し・リード文・本文・問い合わせ先）で作成してください。
```

**⑦ メルマガの件名案**
```
以下のメルマガ内容に合う件名を10案作成してください。
開封率を意識した、読者が思わずクリックしたくなる件名にしてください。

【メルマガの主なテーマ】：
【ターゲット読者】：
【送付時期・季節感】：
件名は25字以内で、数字・疑問形・メリット訴求など複数のアプローチで作成してください。
```

### 人事・採用

**⑧ 求人票の職務内容**
```
あなたは採用経験豊富な人事のプロです。
以下の情報をもとに、求人票の職務内容欄を作成してください。

【職種】：
【主な業務内容（箇条書きで3〜5点）】：
【必須スキル】：
【歓迎スキル】：
【職場環境の特徴】：
応募者の目線で、魅力が伝わる表現にしてください。
```

**⑨ 面接評価コメント**
```
以下の面接メモをもとに、評価コメントを作成してください。

【候補者名】：
【応募職種】：
【面接で確認できた点（良い点）】：
【懸念点】：
【総合評価】：採用 / 見送り / 再考
採用判断の根拠が明確になるよう、具体的な事実に基づいて記載してください。
```

**⑩ 研修資料のアウトライン**
```
以下のテーマで社内研修の資料アウトラインを作成してください。

【研修テーマ】：
【対象者】：
【所要時間】：
【習得してほしいこと（3点）】：
各セクションのタイトル・所要時間・主な内容を含むアウトラインを作成してください。
```

### 企画・経営

**⑪ SWOT分析のフレームワーク**
```
以下の事業・プロジェクトのSWOT分析を行ってください。

【対象】：
【業界・市場環境】：
【主な競合】：
【自社の強み（わかる範囲で）】：

強み・弱み・機会・脅威の各項目を3点ずつ、具体的に挙げてください。
最後に、最も注力すべき戦略の方向性を1段落でまとめてください。
```

**⑫ 会議のアジェンダ作成**
```
以下の会議のアジェンダを作成してください。

【会議の目的・ゴール】：
【参加者】：
【所要時間】：
【決めたいこと・議論したいこと】：

各議題に想定時間を割り当て、最後に「決定事項の確認」の時間を設けてください。
```

**⑬ 議事録の整理**
```
以下の会議メモをもとに、正式な議事録を作成してください。
```

## 07｜Excel・VBA・データ分析への活用

ExcelやGoogleスプレッドシートへの活用は、書籍レビューで最も高く評価されているカテゴリのひとつです。「VBAが書けると知らなかった」という発見の声が多い分野でもあります。

### 関数の作成

**基本的な依頼方法：**

```
以下の処理をExcelの数式で実現してください。

【やりたいこと】：A列に商品名、B列に単価、C列に数量があり、
　　　　　　　　　D列に合計金額を表示したい
【補足条件】：数量が0のときは「-」と表示する
【使用環境】：Excel 2021 / Microsoft 365
```

**よく使う関数の依頼パターン：**
- 「IF関数を使って〜という条件分岐を作りたい」
- 「VLOOKUP（またはXLOOKUP）で別シートのデータを参照したい」
- 「COUNTIF/SUMIFで条件付き集計をしたい」
- 「複数条件を組み合わせたフィルタ処理をしたい」

### VBAマクロの作成

ExcelのVBA（マクロ）はプログラミング知識がなくてもClaudeに依頼できます。

**依頼テンプレート：**

```
ExcelのVBAマクロを作成してください。

【やりたいこと】：
　毎月、売上シートのデータを月別シートにコピーし、
　合計行を追加して、ファイル名に日付を付けて保存する

【ファイル構成】：
　・シート名「売上データ」にA1〜F列でデータあり
　・月別シートは「2026年1月」の形式

【補足】：エラーが起きたときはメッセージを表示してほしい
【環境】：Windows版Excel 2021
```

Claudeが生成したVBAコードは、ExcelのAlt+F11でVBAエディタを開き、そのまま貼り付けて実行できます。

### データ分析・集計

```
以下のデータを分析し、ビジネスインサイトを教えてください。

【データ（CSVまたは表を貼り付け）】：
（データを貼り付け）

【分析の目的】：売上低下の原因を探りたい
【注目してほしい観点】：月別トレンド・商品別・地域別
```

### 注意点

- **生成されたコードは必ずテスト用データで確認する**：本番データに直接適用する前に、コピーしたデータで動作確認を行う
- **エラーが出たらそのままClaudeに貼り付ける**：「以下のエラーが出ました：（エラーメッセージ）」と伝えるだけで修正案を出してくれる
- **大量データの処理は限界がある**：Claudeが直接処理できるデータ量には上限があるため、大規模データはExcelやPythonで処理し、結果の解釈をClaudeに依頼する

この記事で学んだことを5点でまとめると：

1. **役割指定の一行で回答品質が劇的に変わる** — 職種・経験年数・視点を指定する
